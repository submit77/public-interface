# Security

## Supported versions

Only the current default branch is supported.

## Reporting

Use GitHub's private vulnerability reporting for security issues. Do not open a
public issue containing exploit details or sensitive data.

## Trust boundary

This project publishes public information. It is not an authentication or
authorization system. Never place credentials, private drafts, personal records,
deployment tokens, or analytics secrets in content or Git history.

The included headers and Content Security Policy are a baseline, not a substitute
for reviewing a particular deployment.
